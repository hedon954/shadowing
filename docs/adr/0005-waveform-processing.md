# ADR-0005: 波形生成、缓存与渲染

- Status: Proposed
- Date: 2026-07-11

## Context

应用需要展示长 MP3 的完整波形、可拖动选区和实时录音波形。逐帧绘制原始
PCM 会造成内存和渲染压力，长文件每次打开都重新解码也会影响启动体验。

## Decision

- 使用 AVAssetReader 离线读取 PCM，以 min/max 包络保留每个时间桶的正负振幅。
- 缓存多分辨率、带格式版本且与源文件指纹关联的包络数据；不缓存完整解码 PCM。
- 原音与 Take 共用生成和缓存链路，旧版本缓存自动失效并可重建。
- 时间轴根据可见范围和像素宽度选择分辨率，SwiftUI Canvas 每帧只绘制当前
  视口需要的包络点。
- 实时录音在后台写入任务中生成带时间戳的有界包络预览，音频 callback 只做
  有界复制和入队；Take 完成后生成稳定多分辨率缓存。
- Original 与 Take 使用同一时间视口；Take 的本地时间 0 映射到录音选区起点。
- 源文件大小或修改时间变化时使缓存失效。

在长音频性能 Spike 通过前，本 ADR 保持 Proposed。

## Consequences

首次导入需要后台预处理，min/max 数据约为单峰值缓存的两倍；后续打开、选区
对齐和窗口缩放不需要重新解码。视口状态是会话 UI 状态，不写入数据库。

## Verification

- 使用至少 60 分钟 MP3 测量导入耗时、峰值内存和窗口缩放流畅度。
- 连续缩放与平移时，每轨单帧绘制不超过 4096 个包络点。
- 5、30、60 秒选区中 Original 与 Take 的同一相对时间映射到同一横坐标。
- 提前停止的 Take 区间终点与实际录音时长一致，不保留未录制的尾部。
- 生成任务可取消，失败不阻止音频直接播放。
- 缓存删除后可以无数据损失地重建。
